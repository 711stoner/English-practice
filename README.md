# Sentence Memo (React + Vite)

## 学习情况数据

- 学习统计会同步写入 `data/learning_stats.json`（纯文本，适合 GitHub 跟踪）。
- 启动 `npm run dev` 或 `npm run preview` 时，Vite 中间件会自动提供：
  - `GET /api/learning-stats`
  - `POST /api/learning-stats/upsert`
- 同一天会按 `date(YYMMDD)` upsert，不会重复新增多条同日期记录。
- 应用内入口：`仪表盘 -> 学习规则说明`，可直接查看当前学习规则，无需离开应用查看仓库。

## 当前复合科学记忆曲线和背诵规则

本项目是“英文整句默写背诵”，不是单词选择题。系统核心目标是：先保证复习，再控制每日负荷，并在同一天对难句做短程加固。

### 1) 主记忆算法（复合调度）

- 使用 `ts-fsrs` 作为主记忆算法（不是固定天数表）。
- 当前参数：
  - `enable_fuzz: true`
  - `enable_short_term: false`
- 含义：
  - 长期间隔由 FSRS 负责；
  - 会话层保留“同日短程强化”；
  - 对新句和早期成功回忆，再额外套一层 `1天 -> 3天 -> 7天 -> 15天` 的早期巩固上限，成熟后再完全交给 FSRS。

### 2) 评分映射（3档）

- `会（记住了）` -> `Rating.Good`
- `模糊（有点难）` -> `Rating.Hard`
- `不会（忘了）` -> `Rating.Again`

系统在你点评分后才更新 FSRS 卡片、间隔和下次复习时间。

### 3) 拼写确认强度（按熟悉度分层）

- `new / unfamiliar`：保持双重确认（第一轮可模糊通过，第二轮需内容准确；标点/空格/大小写只提示）
- `weak`：一次内容准确可直接评分；若只模糊通过，仍需再确认一次
- `stable`：一次内容正确即可评分（不过若本轮使用辅助，评分会被上限限制）

### 4) 今日练习队列（会话层策略）

- 每日练习主队列上限：`15` 句。
- 队列由两部分组成：
  - 今日待复习（含逾期，优先）
  - 今日允许引入的新学（动态配额）
- 当日新增句子不会立刻算作今日新学，会从次日开始进入新学候选池。
- 新学配额会根据复习负担和学习状态自动收缩/放宽。
- 难句和不熟句会被优先排序。
- `new / unfamiliar / 困难句` 在同一会话内更容易被“3~6题后再次出现”做短程强化。

> 说明：Dashboard 显示的是“总待复习量”；Practice 才执行“每日15句上限”。

### 5) “不会这句”处理

- 不会后不会立刻改长期 FSRS（避免误伤长期调度）；
- 第一次不会：插回后面第2~4题；
- 第二次不会：插回后面第1~3题；
- 同句同轮多次不会会触发“本轮最高按模糊记录”的保护策略，并提高当轮强化优先级。

### 6) 打卡规则

每轮结束后统计：

- `reviewed_count`
- `passed_count`
- `fuzzy_count`
- `failed_count`
- `recall_score = (passed_count + fuzzy_count * 0.6) / reviewed_count`

打卡标准（全部满足）：

- `(passed_count + fuzzy_count) >= 3`
- `recall_score >= 0.45`
- `(passed_count + fuzzy_count) / reviewed_count >= 0.70`
- `failed_count / reviewed_count < 0.30`

按钮状态：

- 达到打卡标准且当天未打卡：`今日打卡`
- 当天已打卡：`今日已打卡`
- 当天主队列做完但未达标：继续巩固到达标（计入本轮）

同一天只能打卡一次。

### 7) mastered（长期掌握）规则

当句子满足以下条件，会标记为已掌握并暂停进入日常复习队列：

- `reps >= 8`
- `intervalDays >= 730`
- 且本次评分为 `模糊` 或 `会`（对应质量 `q >= 3`）

### 8) 学习情况数据落盘（GitHub 友好）

每日学习数据会 upsert 到 `data/learning_stats.json`，核心字段包括：

- `date`（YYMMDD）
- `checked_in` / `checkin_status`
- `new_count`（当天真正进入 Practice 新学并开始练习的句子数；当日新增的句子从次日才可能计入）
- `review_count`
- `pass_count` / `fuzzy_count` / `fail_count`
- `pass_rate`
- `study_seconds`（动作驱动的有效学习时长）
- `created_at` / `updated_at`

### 9) 今日有效学习时长口径

只按“真实学习动作”累计，不按页面打开时长累计：

- 触发动作示例：输入、提交、评分、下一句、查看提示等学习行为
- 采用“相邻动作间隔累计 + 单次最多30秒”规则
- 页面失焦/切后台会暂停
- 因此不会把挂机时长误算成学习时长

### 10) 模糊通过与辅助上限规则

- 核心判定会忽略大小写与标点（用于判断是否通过/进入确认/进入评分）。
- 若只是空格分词不规范，也不单独拦截，会给轻提醒。
- 大小写与标点问题只做轻提醒，不作为答错阻断。
- 模糊判定按句长分层：短句更严格，长句允许少量非核心误差（但核心结构错误不放行）。
- 若本轮用过提示 / 朗读 / 不会这句，再给“会（Good）”会被上限截断为“模糊（Hard）”，以保持 FSRS 输入更干净。
- 一旦本轮只能记为 `Hard`，后续间隔也会按更短的早期巩固阶段处理，而不是直接放大到长间隔。
